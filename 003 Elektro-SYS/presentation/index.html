Mechanism Design summed up!

  • "agents" have private information.
  • which game – do you have to let agents play – to get the outcome that you want?

Social choice function

Desired outcome as a function of the agents’ types

\[ f : \underbrace{\Theta_1 \times \Theta_2 \times ... \times \Theta_I}_{\text{type set of agents}} \quad \to \underbrace{X}_{\text{set of alternatives}} \]

Mechanism

\[ \Gamma = \left( \underbrace{S_1, S_2, ..., S_I}_{\text{strategy sets}} \quad, \underbrace{g(\cdot)}_{\text{outcome function}} \right) \]

where \(g : S_1, S_2, ..., S_I \to X\)

Mechanism

\[ \Gamma = \left( \underbrace{S_1, S_2, ..., S_I}_{\text{strategy sets}} \quad, \underbrace{g(\cdot)}_{\text{outcome function}} \right) \]

where \(g : S_1, S_2, ..., S_I \to X\)

Induces a Bayesian game:
  • Agents choose a strategy from their strategy set.
  • Through the mechanism (i.e. rules of the game) their payoff depends on all agents’ strategy choices.
  • Every agent is uncertain about other agents’ type.

Implementing a social choice function

The mechanism \(\Gamma\) implements the social choice function \(f\),
if the game has an equilibrium in which we get the desired outcome, i.e.: \( g \left( s^*_1(\theta_1), s^*_2(\theta_2), ..., s^*_I(\theta_I) \right) = f(\theta_1,\theta_2,...,\theta_I) \) \(\quad\forall \theta_i, i\)

Revelation principle

if \(f\) can be implemented by some mechanism \(Γ\),
then \(f\) is truthfully implementable

\[ \Updownarrow \]

if truthful implementation does not work,
then no mechanism will work

Equilibrium concepts

Dominat strategy equilibrium:, i.e. \(s_i\) that maximises utility, no matter what other agents do.\(\quad\forall \theta_i, i\)

  • No need to predict what others will play.
  • No need to know type distribution.
  • Works even if agents think others are non-rational/ill-informed.

So, which social choice functions are (truthfully) implementable (in dominant strategies)?

Equilibrium concepts

Bayesian Nash equilibrium:, i.e. if no type of no agent can increase expected payoff by deviating.

  • More implementable social choice functions.
  • Less confident that agents adhere to it in real-life.
  • Agents and "mechanism designer" needs to know type distribution.

Extending Groves-Clarke ...

Assume that agents' types are statistically independent.

Expected Externality mechanism: Agents pay expected externality imposed on others, while all collected payments are recirculated among agents.
  • Trutfully implementable in Bayesian Nash equilibrium
  • Efficient (allocation)
  • Budget balanced
  • However, does not fulfill participation constraint
    (agents pays more than their valuation – will not participate voluntarily).

Is there a trading mechanism with efficient (allocation), balanced budget, and where agents will participate voluntarily?

Tension between incentives and efficiency. I.e. no matter the mechanism – agents are going to tend to lie – leading to some inefficient trades.

However

  • inefficiencies decrease asymtotically as the number of buyers and sellers increase.
  • the theorem, concerns ex post efficiency (i.e. when all types are know).
    Many mechanisms are ex ante effcient (i.e. before knowing the types) or interim effcient (i.e. when knowing own type but not others' types) – so timing matters.

Optimal mechanisms

Often take the perspective of one agent and choose the effcient (and feasible) mechanism that maximizes this agent’s utility.

  • pricing scheme that maximizes the profits of a seller
  • regulatory scheme that maximizes consumer surplus
  • income tax scheme that maximizes the payoff of a utilitarian government
  • etc.

Private information creates distortions – you have to pay rents in order to truthfully extract information – and distortions can reduce these information rents.
